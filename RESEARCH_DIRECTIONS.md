# EEGTokenizer 研究方向

罗列所有可能的研究方向组合，按优先级排序。

---

## 核心问题

1. **Tokenizer 设计**：如何设计更好的 EEG tokenizer？
2. **性能验证**：如何验证 tokenizer 的好坏？

---

## 一、Tokenizer 设计方向

### 1.1 时空频编码（当前方案）
**方案**：当前 EEGSTFEncoder 的方案
- 时间分patch
- 5分支节律卷积
- 空洞卷积空间建模
- 相似patch合并

**优点**：
- 有明确的神经生理学依据（节律 bands）
- 空间建模考虑了电极位置关系
- 已有基础代码实现

**缺点**：
- 计算复杂度较高
- 超参数多（卷积核、膨胀率等）
- 缺乏理论保证

**优先级**：P0（作为基线）

---

### 1.2 ADC 风格数字量化（老板新思路）
**方案**：固定时间分割 → ADC 风格量化 → 码字聚合

#### 子方向组合

| 维度 | 选项 | 说明 |
|------|------|------|
| **时间分割** | 固定长度 | 250Hz × 0.25s = 62.5ms → ~16个patch/秒 |
| | 事件触发 | 基于 EEG 事件（如峰谷值）分割 |
| | 自适应长度 | 根据信号复杂度动态调整 |
| **量化方式** | 标量量化 | 每个通道独立量化（类似 ADC） |
| | 矢量量化 | 多通道联合量化（VQ/VQ-VAE） |
| | 乘积量化 | 高维矢量拆分后量化（PQ） |
| **量化精度** | 1bit | 二值化（0/1） |
| | 2bit | 4级 |
| | 4bit | 16级 |
| | 8bit | 256级 |
| | 16bit | 65536级（全精度） |
| **码字聚合** | mean/sum | 简单平均/求和 |
| | 注意力聚合 | 学习权重 |
| | 门控聚合 | LSTM/GRU 门控 |
| | 聚类聚合 | 码字聚类后聚合 |

#### 优先级排序

**P0（核心探索）：**
1. 固定长度 + 标量量化 + 4bit + mean聚合
2. 固定长度 + 标量量化 + 8bit + 注意力聚合

**P1（次优先）：**
3. 固定长度 + 矢量量化 + 4bit + 聚类聚合
4. 自适应长度 + 标量量化 + 8bit + mean聚合

**P2（探索性）：**
5. 事件触发 + 乘积量化 + 4bit + 门控聚合
6. 各种组合的 ablation study

---

### 1.3 其他 Tokenizer 思路

#### 1.3.1 频域字典学习
**方案**：学习一个频域字典，将 EEG 投影到字典上作为 token

**优点**：可解释性强（字典原子对应特定节律）
**缺点**：需要大量数据学习字典
**优先级**：P1

#### 1.3.2 小波变换 Tokenization
**方案**：用小波变换替代 FFT，做多尺度时频分析

**优点**：时频局部化更好
**缺点**：计算量大，基函数选择难
**优先级**：P2

#### 1.3.3 端到端可学习 Tokenization
**方案**：用 VQ-VAE 或类似架构，端到端学习 tokenizer

**优点**：自动学习最优表示
**缺点**：可解释性弱，训练不稳定
**优先级**：P1

## 二、性能验证方向

### 2.1 重构质量（直接指标）
**方案**：Tokenize → Detokenize，计算恢复误差

**指标**：
- MSE（均方误差）
- MAE（平均绝对误差）
- SNR（信噪比）
- 频域误差（各节律 band 的恢复误差）

**优点**：直接、客观、可解释
**缺点**：需要设计合理的 detokenizer
**优先级**：P0

---

### 2.2 探针任务（表示质量）
**方案**：冻结 tokenizer，在中间表示上训练简单探针

**探针任务**：
- 节律分类（delta/theta/alpha/beta/gamma）
- 运动想象分类（BCI IV 2a）
- 被试识别（验证表示的个体特异性）
- 噪声鲁棒性（加噪后表示稳定性）

**优点**：可以评估表示的不同方面
**缺点**：探针性能受探针模型影响
**优先级**：P0

---

### 2.3 可视化（可解释性）
**方案**：可视化 token 表示

**方法**：
- t-SNE/UMAP 降维可视化
- 码字分布统计
- 相似 EEG 片段的 token 相似度
- 空间拓扑可视化（电极位置 + token 相似性）

**优点**：直观、可解释
**缺点**：定性为主，缺乏定量指标
**优先级**：P1

---

### 2.4 对比学习（表示质量）
**方案**：设计对比学习任务，评估表示质量

**对比任务**：
- 同一试次的不同片段 → 相似
- 不同试次的同一类 → 相似
- 不同被试的同一类 → 相似
- 加噪前后 → 相似

**指标**：
- InfoNCE loss
- Recall@k
- 平均相似度

**优点**：不需要标签，可大规模预训练
**缺点**：对比任务设计难
**优先级**：P1

---

### 2.5 迁移学习（泛化性）
**方案**：在数据集 A 上预训练 tokenizer，在数据集 B 上 finetune

**迁移场景**：
- 跨被试迁移
- 跨任务迁移（运动想象 → 情绪识别）
- 跨设备迁移（不同 EEG 设备）
- 跨范式迁移（事件相关 → 稳态视觉诱发电位）

**优点**：验证 tokenizer 的泛化能力
**缺点**：需要多数据集
**优先级**：P2

---

### 2.6 下游任务（间接指标，当前方案）
**方案**：在下游任务（如运动想象分类）上端到端训练

**指标**：
- Accuracy
- Precision/Recall/F1
- AUC

**优点**：直接面向应用
**缺点**：耦合度高，难以归因
**优先级**：P0（作为基线参考）

---

## 三、研究路线图

### Phase 1: 基线建立（P0）
1. 实现当前时空频 tokenizer 的完整训练 pipeline
2. 建立下游任务（BCI IV 2a）基线
3. 实现重构质量评估框架

### Phase 2: ADC 风格量化探索（P0）
1. 实现固定时间分割 + 标量量化（1/2/4/8/16bit）
2. 实现简单聚合（mean/sum）和注意力聚合
3. 在重构质量和探针任务上评估
4. 做 ablation study（精度、聚合方式）

### Phase 3: 多维度探索（P1）
1. 探索不同时间分割方式（事件触发、自适应）
2. 探索矢量量化、乘积量化
3. 实现对比学习评估
4. 可视化分析

### Phase 4: 高级探索（P2）
1. 迁移学习实验
2. 其他 tokenizer 思路（小波、字典学习）
3. 码字聚类聚合等高级聚合方式

---

## 四、Quick Win 候选

1. **固定长度 + 4bit + mean** - 实现简单，可快速验证思路
2. **重构质量评估框架** - 可快速建立直接评估指标
3. **探针任务（节律分类）** - 简单，可快速评估表示质量
